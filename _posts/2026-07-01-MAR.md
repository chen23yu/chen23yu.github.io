## 从VQ-VAE,到VQ-GAN,再到MaskGIT(待补全)

![从VQ-VAE到MAR](/assets/images/posts/mar-overview.png)

### VQ-VAE：
训练时，输入一张图像x0,经过encoder编码成一个连续的向量，然后利用embedding

## 先明确 MaskGIT 遗留的真正问题是什么
### 第一个障碍：**丢掉 codebook 之后，拿什么衡量"这个位置我预测得有多有把握"？**
你已经理解 MaskGIT 的推理逻辑：全 mask 开始，每轮预测所有被遮位置的概率分布，取置信度最高的一批"锁定"，剩下的重新 mask，迭代几轮完成。

这套机制能跑起来，**关键依赖一件事：每个位置的预测输出是一个离散概率分布（softmax over codebook，比如 8192 类）**。有了概率分布，你才能算"置信度"（比如取最大概率值，或者用这个分布算熵），才能排序、才能决定"这轮该锁定谁"。

现在问题来了：如果不想要离散 codebook，想让每个位置直接预测一个连续向量（比如 768 维的浮点数），**"置信度"这个概念还存在吗？** 一个连续向量没有天然的"概率最大值"这种东西——你不能对连续输出做 softmax。

这是从 MaskGIT 走向 MAR 必须解决的第一个障碍：**丢掉 codebook 之后，拿什么衡量"这个位置我预测得有多有把握"？**

### 第二个障碍：连续值怎么算损失、怎么采样

离散 token 预测用交叉熵天然合适——目标就是"猜哪个类"。连续向量呢？如果直接用 L2 loss 让模型回归到目标向量，会出现你之前问 VQ-VAE 时提到的那个老问题：**多个合理答案的平均值是模糊的**。比如某个 masked 位置，上下文允许它是"红色"或"蓝色"两种同样合理的语义，L2 回归会输出"红蓝之间的某种颜色"——一个哪个都不是的错误答案。

所以第二个必须解决的问题是：**怎么让模型对连续目标做出"多峰"预测，而不是被 L2 拍扁成一个模糊平均值？**

### MAR 的解法：用一个小扩散模型代替 softmax

MAR 对这两个障碍给出了同一个答案——**在每个位置挂一个微型扩散模型（diffusion head）**。

先看第二个障碍是如何解决的。扩散模型本身就是为解决 "多峰分布" 问题而设计（这也是它能够生成多样化图像，而非每次输出同一张 "平均脸" 的原因）。

具体流程：该位置的真实目标为向量$x_i$，训练时向其添加噪声得到带噪向量$x_i^t$；扩散头（diffusion head）接收 Transformer 对该位置的表征（即隐状态$z_i$，它是 MAR 框架内的条件信息，作用与 MaskGIT 中 "用于预测对应位置的上下文表征" 完全等价），结合$x_i^t$，预测添加进去的噪声。

训练完成后，推理阶段从纯噪声起步，通过这个小型扩散头执行多轮去噪，即可采样得到清晰、确定的向量。同时由于扩散过程自带随机性，在相同上下文下多次采样能够产出多个合理但互不相同的结果，天然保留了多峰分布的特性。

**关键理解点**：这个 diffusion head 极小（通常只有几层 MLP），而且是**逐位置独立运行的**——它不是"再跑一个大扩散模型生成整张图"，而是"对 Transformer 已经算好的某一个位置的 hidden state，用几步小去噪把这个位置该填的向量具体值采样出来"。所以它的开销很低，可以在 MaskGIT 式的多轮迭代里塞进每一步而不显著拖慢速度。

### 第一个障碍怎么解决——置信度这个概念被彻底放弃了

这一点是很多人在这个过渡处卡住的地方，值得单独讲清楚。

MAR 没有想办法给连续预测"造一个置信度出来"，而是**换了一种决定"这轮该解锁哪些位置"的策略**：与其按置信度排序，不如**按随机顺序**（或者论文里叫"random order"）逐步解锁——每一轮随机挑选一批还没解锁的位置，用 diffusion head 把它们的值采样出来，就当作确定了，纳入下一轮的可见上下文。

这也是 MAR 论文题目里 **"Autoregressive"** 这个词的来源——它本质上仍然是一种自回归过程（一步步把不确定变成确定，后面的预测依赖前面已确定的），只是**生成顺序从"从左到右"（经典自回归）或者"按置信度排序"（MaskGIT）换成了"随机顺序"**，同时每一步可以同时确定一批（而不是一个）位置，所以仍然保留了并行加速的优势。

### 把三者的"迭代解锁策略"放在一起对比，你会看得更清楚

**自回归（VQGAN-Transformer）**：顺序固定为从左到右，每步只解锁 1 个，靠的是训练时天然的"从左到右建模"这个顺序假设。

**MaskGIT**：顺序由置信度动态决定，每步解锁一批（置信度最高的那些），依赖的是 softmax 输出的置信度分数。

**MAR**：顺序是随机的，每步解锁一批（随机选定数量的位置），因为没有置信度可用，干脆放弃"挑最有把握的"这个策略，改成"随机挑一批，用扩散头认真采样出结果"。

这个转变背后的逻辑其实是：既然扩散头本身已经能针对任意上下文给出高质量、多峰的采样结果，那么"先挑简单的、有把握的"这件事就不再像 MaskGIT 里那么重要了——因为哪怕上下文信息不完整，扩散头依然能给出一个合理的（虽然带有不确定性的）采样结果，不需要靠置信度筛选来规避"预测不准"的风险。

### 完整走一遍 MAR 的训练+推理流程，跟 MaskGIT 逐项对比

**训练阶段**（对比 MaskGIT）：

MaskGIT 训练时：随机选一部分位置 mask 掉，模型对每个被 mask 位置输出一个 softmax 分布，用交叉熵监督，让分布逼近真实的离散 token。

MAR 训练时：同样随机选一部分位置 mask 掉（这一步逻辑完全一致），但模型对每个被 mask 位置输出的不是分布，而是一个 hidden state $z_i$，这个 $z_i$ 被喂给 diffusion head，diffusion head 用扩散损失学习"给定 $z_i$，如何从噪声还原出真实向量 $x_i$"。

**推理阶段**（对比 MaskGIT）：

MaskGIT 推理：全 mask 开始 → 模型预测所有位置的分布 → 按置信度锁定一批 → 重复几轮。

MAR 推理：全 mask 开始 → 模型对所有未解锁位置输出 hidden state → **随机挑选**一批位置 → 对这批位置分别跑 diffusion head 的几步去噪采样，得到具体向量值 → 这批位置视为"已确定"，纳入下一轮可见上下文 → 重复几轮，直到所有位置都被采样出来。

### 一句话总结这次过渡

**MaskGIT 到 MAR 的过渡，本质是把"填空框架"里"怎么衡量预测质量、怎么决定解锁顺序、怎么表达预测结果"这三件事，从"离散分类 + 置信度排序"整体替换成了"连续扩散采样 + 随机顺序"。** 框架的骨架（mask 一部分、模型看可见部分预测被遮部分、迭代多轮逐步补全）完全没变，变的是每一步"预测"这个动作背后的数学工具。

回到 Bernini：现在你再看它的规划器就会发现，它其实精确复刻了 MAR 的这套逻辑——MLLM 就是那个输出 hidden state 的大 Transformer，ViT Embedding Decoder 就是那个逐位置的小 diffusion head（只不过用的是流匹配而不是原始 DDPM 形式），推理时 25 步迭代解码走的也是 MAR 这套"随机/预定顺序解锁 + 扩散采样"的框架，而不是 MaskGIT 那套置信度排序。





# MAR 架构详解

完全抛开对比，只讲 MAR 自己的架构，一个模块一个模块讲清楚。

## MAR 要解决的任务

给定一张图，把它变成一组向量，随机挖掉一部分，训练一个系统去补全被挖掉的部分。这个系统由三个独立的模块组成，我们一个一个看。

## 模块一：Tokenizer（连续 VAE）

这是一个提前训练好的、固定不变的模块（训练 MAR 主体时它是冻结的，不参与训练）。

```
输入：一张图像，比如 256×256×3
输出：一个 16×16 的网格，网格里每个格子是一个 16 维的浮点向量（不是整数，不查表）
```

所以一张图变成了 256 个连续向量（16×16=256 个位置）。这一步跟 codebook、量化毫无关系，就是普通的自编码器压缩。

## 模块二：主干 Transformer（论文里叫 encoder-decoder 或 backbone）

这是 MAR 真正训练的核心大模型。

**输入**：刚才那 256 个位置的向量网格，但其中一部分位置（比如挖掉 70%）被替换成一个**统一的、可学习的 mask 占位向量**（跟这些位置原本的连续向量维度一样，只是这个位置的值被换成了一个特殊的、训练过程中学出来的固定向量，类似 BERT 里的 `[MASK]` token embedding，只不过这里是连续向量版本）。

```
位置1: 真实向量 v1（未被遮）
位置2: [MASK]向量（被遮）
位置3: 真实向量 v3（未被遮）
位置4: [MASK]向量（被遮）
...
位置256: 真实向量 v256（未被遮）
```

**内部处理**：这一整条序列（256 个位置，不管遮没遮都在里面）送进一个标准 Transformer（多层 self-attention + FFN，跟 BERT 内部结构一样），每个位置都能相互看到彼此（双向注意力，没有因果 mask 限制）。

**输出**：Transformer 对每一个位置都吐出一个输出向量。**注意，这个输出向量的维度和含义，跟输入的图像向量完全不是一回事**——它是 Transformer 这一层网络算出来的一个隐藏表示（hidden state），你可以理解成"模型对这个位置内容的语义总结/猜测方向"，但它本身还不是"这个位置最终应该填的具体数值"。

到这里为止，主干 Transformer 的工作就结束了。它只对被 mask 的那些位置，各自产出一个 hidden state：$z_1, z_2, \ldots$（每个被遮位置一个）。

## 模块三：Diffusion Loss / Diffusion Head

这是 MAR 最独特的部分，也是最小的一个模块（通常只有 3 层左右的小 MLP，参数量远小于主干 Transformer）。

**它是干什么用的**：它的任务只有一个——**对每一个被 mask 的位置，把该位置的 hidden state $z_i$ 转换成一个具体的、真实数值意义上的目标向量**（也就是模块一 Tokenizer 输出的那种 16 维连续向量，跟原图对应位置的向量维度完全一致）。

**训练时它怎么工作**（这是关键，你需要盯住这个流程）：

对某个被 mask 的位置 $i$，它真实对应的目标向量是 $x_i$（模块一 Tokenizer 编码出来的、这个位置原本该有的值，训练时是已知的，用来当 ground truth）。

```
第1步：拿到真实目标 x_i
第2步：随机生成一点噪声 ε，按扩散模型标准做法给 x_i 加噪，得到 x_i^t（t是某个随机的噪声程度）
第3步：把 x_i^t 和 z_i（主干Transformer给这个位置的hidden state，当作条件）一起喂进这个小MLP（diffusion head）
第4步：小MLP输出一个预测，去预测"刚才加进去的噪声ε是什么"
第5步：算损失 = ||真实噪声ε - MLP预测的噪声||^2，反向传播更新参数
```

这跟你熟悉的标准扩散模型（比如 Stable Diffusion 的 U-Net 学"预测噪声"）**训练方式完全一样**，唯一区别是这里的"图像"变成了一个 16 维小向量，"U-Net"变成了一个几层的小 MLP，规模小了非常多，但训练原理（加噪声、学着预测噪声、用 MSE 损失）没有任何变化。

**整个 diffusion head 是所有位置共享同一套参数的**——不是每个位置单独训练一个 MLP，而是同一个小 MLP，被反复喂进不同位置的 $(x_i^t, z_i)$ 去训练，学会"给定任意一种 $z_i$ 语境，如何做去噪"这件通用的事。

## 推理时（生成阶段），三个模块怎么串起来跑

```
第1步：所有256个位置全部设为[MASK]占位向量，喂进主干Transformer
第2步：主干Transformer对每个位置输出一个 z_i
第3步：随机挑一批位置（比如16个），准备"确定"它们的值
第4步：对这16个位置，各自拿着自己的 z_i，让diffusion head从纯随机噪声开始，
       跑标准扩散模型的采样流程（比如50~100步小去噪迭代），
       最终得到这16个位置各自的具体连续向量 x̂_i
第5步：把这16个位置从[MASK]换成刚采样出来的具体向量，其余仍然是[MASK]
第6步：把更新后的256个位置（16个已确定+240个仍mask）重新喂进主干Transformer
第7步：重复第2~6步，每轮解锁更多位置（比如16个→32个→64个……），大约十几轮后全部256个位置都有了具体值
第8步：256个位置的向量网格 → 模块一的Decoder（VAE decoder） → 还原成最终图像
```

## 一句话总结 MAR 的架构本质

**MAR = 一个双向 Transformer（负责读懂上下文、对每个空位给出一个语义提示 $z_i$）+ 一个共享参数的小型扩散模型（负责把每个提示 $z_i$ 具体地"画"成一个真实的连续向量）。** 这两个模块是级联关系，不是二选一——Transformer 自己完全不产出最终答案，它产出的 $z_i$ 只是喂给下游 diffusion head 的条件输入，真正"生成具体数值"这件事，从头到尾都是 diffusion head 在做。
