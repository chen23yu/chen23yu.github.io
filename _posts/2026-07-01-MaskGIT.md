MaskGIT（Chang et al., CVPR 2022）的出发点很简单：**自回归从左到右一个个生成太慢了，能不能一次生成多个 token？**

![MaskGIT概述](/assets/images/posts/maskgit-overview.png)

关键点是：BERT 在预测"很"这个位置时，**同时能看到左边的"今天天气"和右边的"好，适合[MASK]步"**——双向信息都能利用。这比只能看左边的自回归模型（GPT 风格）信息更充分。

MaskGIT 把这个思路搬到图像 token 上：


**训练阶段**：随机遮住图像 token 网格里的一部分，让 Transformer 从可见 token 预测被遮住的 token。mask 率从 0% 到 100% 随机采样，模型必须适应各种程度的残缺输入。这样训练出来的模型，给它任何"部分可见"的 token 网格，它都能预测缺失的部分。

**推理阶段**才是 MaskGIT 真正的创新。它不是从左到右一个个生成，而是这样做的：

```
第 1 步：256 个位置全部是 [MASK]，模型一次性预测所有 256 个位置 每个位置得到一个预测 token 和**一个置信度分数**

第 2 步：保留置信度最高的 64 个（比如），这 64 个位置"锁定" 其余 192 个位置重新 mask 住

第 3 步：把 64 个已锁定 token + 192 个 [MASK] 一起输入模型 模型再次一次性预测所有 192 个 masked 位置 保留新一轮置信度最高的 64 个

……重复 8 步，每步解锁更多 token，最终全部填满
```

这个过程有一个余弦调度：前几步解锁少量 token（最有把握的），后几步快速解锁大量 token（因为已经有很多上下文了，剩余 token 也更容易预测）。8 步就能生成 256 个 token，对比自回归需要 256 步，**快了约 32 倍**，而且质量不差甚至更好，因为每次预测都能看到双向上下文。

到这里 MaskGIT 的逻辑已经完整了。但它继承了 VQGAN 的离散化问题——每个 token 必须是 codebook 里有限个向量之一，细节损失有上限
